## 经验
遇到 ANR 位于 SplashActivity 的问题，优先确认 SplashActivty onPause 到 onStop 之间主线程是否有执行耗时操作。


## 概念
1、5、15分钟内正在使用和等待使用CPU+的活动进程的平均数
```
10-19 20:13:06.642  1000   640  3022 E ActivityManager: Load: 4.68 / 1.08 / 0.36
```

表示日志 dump 的是 ANR 发生后的 0ms ~ 7217ms 之间的系统状态，同时也说明了 ANR 发生的时间是 2023-10-19 20:12:59.399
```
10-19 20:13:06.642  1000   640  3022 E ActivityManager: CPU usage from 0ms to 7217ms later (2023-10-19 20:12:59.399 to 2023-10-19 20:13:06.616):
```

中间部分是各个进程 CPU 占用情况

最后一行是所有进程 CPU 占用合计

## ANR 类型
输入事件超时(5s) 
```
10-19 20:13:06.642  1000   640  3022 E ActivityManager: Reason: Input dispatching timed out
```

## 原因分析
初始化 services 完成后，SplashActivity 进入 onPause 状态，并跳转到 HomePageActivity，等待 HomePageActivity 执行生命周期 onCreate, onStart, onResume，但由于 HomePageActivity 在生命周期函数中执行了过多的耗时逻辑代码，导致 SplashActivity 迟迟无法进入 onStop 和 onDestory，引发了超时。

`20:12:54.272` SplashActivity 进入 onPause，同时 HomePageActivity 也进入 onCreate 状态
```
10-19 20:12:54.272  1000  1254  1254 D SplashPresenter: onPause
10-19 20:12:54.300  1000  1254  1254 D HomePageActivity: onCreate
```

一直到系统触发 ANR 时，liveplayer 的主线程还在执行 HomePageActivity 的逻辑代码
```
10-19 20:12:59.316  1000   640   755 I WindowManager: Input event dispatching timed out sending to com.intellyva.liveplayer/com.intellyva.liveplayer.ui.SplashActivity.  Reason: 6b300e2 com.intellyva.liveplayer/com.intellyva.liveplayer.ui.SplashActivity (server) is not responding. Waited 5030ms for FocusEvent(hasFocus=true)
10-19 20:12:59.399  1000   640   755 D WindowManager: notifyANR took 84ms
```

又经过 2s 后，SplashActivity 的生命周期才得到释放
```
10-19 20:13:01.105  1000  1254  1254 D SplashPresenter: onStop
10-19 20:13:01.133  1000  1254  1254 D SplashPresenter: onDestroy
```

所以核心是看 HomePageActivity 初始化这段时间主线程在做什么，目前分析下来主要耗时在初始化 WebView，请Donny哥继续帮忙追查，[Bug链接](http://intellyva-win/zentao/bug-view-2464.html)
